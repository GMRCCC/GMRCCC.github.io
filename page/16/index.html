3.2 为超参数选择合适的范围

发表于 2023-10-26 分类于 吴恩达深度学习2-改善深层神经网络

随机取值可以提升搜索效率，但随机取值并不是在有效值范围内随机均匀取值，而是要选择合适的步进值。

我们先来看均匀取值的样子，假设我们需要给某层的单元个数和层数取值

我们可以随机均匀取值，看起来很合理。

如果我们对α进行取值，假设α的取值范围是0.001到1，如果随机均匀取值，那么90%的取值都会落在0.1到1之间，只有10%落在0.001到0.1之间，这显然不太合理

我们可以采用取对数的方法，让α在对数轴中随机均匀取值，在程序中体现就是r均匀取值，α=10^r

β的正常取值是0.9-0.999之间，同样的，当β接近0.999时，会对数值的变化更加敏感(很正常，当β的值接近1时，一点点数值的变化都会让1/(1-β)急剧增长)，由于0.9-0.999不好取值，因此我们取1-β=0.1-0.001，这样β的处理方式就和α一样了

3.3 超参数训练的实践 Pandas vs Caviar

发表于 2023-10-26 分类于 吴恩达深度学习2-改善深层神经网络

根据CPU、GPU的计算资源的不同，训练模型的方式分为Pandas和Caviar两种。

在计算资源缺乏时，人们通常用Pandas方法，即只照看一个或几个模型，然后每天根据它的训练效果去不断调整它的超参数，一直重复照料直到达到预期效果为止

和熊猫类似，一生只有一个孩子，精心照料希望它茁壮成长。

计算资源丰富时，用Caviar方法，即一次性训练多个模型，不人为照看，训练一段时间后取其中效果最好的一个

这种方法类似于鱼产卵，一次性产出很多的卵，希望其中的一个有出色的表现。

3.6 Batch-Norm 为什么会奏效

发表于 2023-10-26 分类于 吴恩达深度学习2-改善深层神经网络

一个原因和归一化输入类似，batch-norm可以使Z的范围相似，加速梯度下降的过程。

另一个原因是covariate shift，如果你已经学习了X到Y的映射，然后改变X的分布，那么你需要重新训练你的学习算法

在深层神经网络中，输入样本的变化会导致不同层的分布不断变化，造成covariate shift问题，通过batch归一化固定方差和均值，一定程度上可以减弱分布的变化。

可以理解为batch-size使每一层的输出值更加稳定，这样当后一层在计算时，可以不那么考虑前面几层网络改变带来的分布的改变，减少了各个层之间参数的联系，变相加速了神经网络的实现。

batch-size归一化在一定程度上稳定了前面层的变化(因为它们被均值和方差所限制)，这会使后层的学习工作变得更加容易。

batch-size也有轻微的正则化效果，进行归一化的过程其实就相当于给输出值加了噪音(乘性噪音和加性噪音)，这使得各个层之间稍微独立了一点点，不过分依赖于前面层的结果，稍微起到了正则化的效果。

如果mini-batch size变大，那么batch-size归一化的正则化效果会减弱。

最后，不要把batch-size归一化当做正则化的手段，它只是一个加速学习的手段，轻微的正则化只是一个附带的效果。

3.9 训练一个Softmax分类器

发表于 2023-10-26 分类于 吴恩达深度学习2-改善深层神经网络

softmax和hardmax对应，hardmax指的是取向量中的最大值为1，其他为0(这种处理方式很”硬”)，而softmax相对柔和一点，算每种类别的概率值

softmax相比于逻辑回归将分类个数从2个推广到了C(C≥2)个，当C=2时，softmax又退化到了逻辑回归

大体的证明思路就是，如果分类个数为2，则最后算出的a[l]是一个二维向量，由于概率和为1，实际上我们只需要算其中的一个数即可，这就又回到了逻辑回归的计算方式。

在定义softmax分类器的损失函数时，我们使用交叉熵来定义单个样本的损失，则成本函数J即为所有样本损失的平均

假设样本为[0 1 0 0]^T,可以看到只有分类器得出第二类的值越大，预测结果才会越准，损失函数才会越低，这表明交叉熵是满足让成本降低的目标的。

对softmax的梯度下降过程，正向传播的计算和前面学的一样，而对于反向传播，你可以自己去求导数，但如果你用了深度学习框架，当你指定正向传播的计算过程时，它会自动帮你计算反向传播
