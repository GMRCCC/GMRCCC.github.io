2.2 理解mini-batch梯度下降法

发表于 2023-10-26 分类于 吴恩达深度学习2-改善深层神经网络

正常的梯度下降法会看到梯度不断的下降，但对mini-batch梯度下降法来说，它的大方向可能是下降的，但会有更多的噪声

这是因为划分了数据集后，每次迭代训练的都是不同的训练集，所以会导致成本函数的波动。

但总体趋势应该是下降的。

数据集的划分有两个极端，一个是完全不划分，另一个是对每个样本都划分。

第一个就是我们前面学的batch梯度下降法，第二个叫做随机梯度下降法。

随机梯度下降法有两个缺点，一个是由于每次只训练单个样本，因此向量化带来的效率优势完全用不到，另一个是训练不能完全保证朝着全局最小值的方向前进，样本的差异导致梯度下降的方向不同。

随机梯度下降法还会导致损失在最小值附近波动。

batch梯度下降则会因为向量矩阵过大，导致需要非常高的算力才能进行计算。

综合来看，mini-batch的效果比较好，即利用了向量化的效率优势，又让梯度下降的整体趋势朝着全局最小值方向

那么平均了多少天是怎么来的呢？假设β=0.9，我们想让它的指数接近自然对数1/e，由极限求和公式可知，这个指数为1/(1-β)=10

我们认为当系数小于1/e时的温度可以忽略(急剧衰减)，即计算的是前1/(1-β)天，当β=0.98时，为50天。注意这不是严格的数学证明。

指数加权移动平均的实现非常简单，我们只需要知道每天的温度θt，设置一个参数，就可以让V实现迭代

相比于直接计算前n天的温度平均值，这样做不仅节省了内存空间(不用保存之前的温度数据)，而且提高了效率，就连实现也只需要一行代码即可。

2.6 动量梯度下降法

发表于 2023-10-26 分类于 吴恩达深度学习2-改善深层神经网络

在传统梯度下降法中，我们需要计算多步来使目标值收敛

由于梯度下降过程中不是总是朝着全局最优方向前进，因此会有一定幅度的摆动。因此，我们需要控制学习率在一个比较小的值，值过大会导致摆动幅度过大，使梯度无法朝着目标值前进，如紫线所示。

但学习率过小会导致收敛的速度特别慢，导致我们需要计算的时间过长。

所以用momentum梯度下降法，即用指数加权移动平均来更新dW值。

通过平均，在梯度下降过程中纵向来回摆的幅度被平均掉了，由于横向都朝着梯度下降的方向，因此横向下降的速度并无太大变化。

通过这种方式，即使采用较大的学习率，也不用担心摆动的幅度过大，保证梯度朝着全局最优方向收敛，如下图，红线表示用momentum法的下降趋势

具体过程类似这样，此时W不再用dW来更新，而是用Vdw来更新

我们可以用球从碗中下降来类比这个过程，假设碗的低端是目标值，球在向下滚，则vdw相当于球的速度，dW相当于球的加速度，加速度会影响球的速度，但不会直接影响球向哪个方向前进，由于有摩擦力的存在，球的速度也不会无限加大。也就是dW此时不会直接影响梯度下降的方向。

β的常见选择是0.9.

在实际使用时我们通常不使用偏差修正，即vdw和vdb的值初始化为0.在有些资料中，momentum梯度下降通常表现为右边紫线的形式，其实就相当于是整体除了1-β，此时vdW的值会比右边有所增大，相应的学习率也要有所变化。

通常左边的形式更加顺眼，如果用右边的形式，当你尝试去改变β值时，α的值也要相应改变。

全称是 root mean square prop，也可以用来加速梯度下降。

为了防止摆动幅度过大，我们在更新参数时除以均方根来对参数进行扼制。如果摆动幅度过大，则除均方根会让摆动幅度变小，如果横向移动速率过小，则除均方根会提高移动速率。

本质就是除均方根来平衡摆动幅度和下降速度，有了这个限制，我们就可以适当地提高学习率来加快迭代速度。

在实际操练过程中，为了防止均方根接近0导致值过大，我们往往会加一个小的数作为后缀来保证数组稳定

momentum和rmsprop梯度下降法都可以降低梯度下降中的摆动，并允许使用一个更大的学习率α来加快学习速度

2.9 学习率衰减

发表于 2023-10-26 分类于 吴恩达深度学习2-改善深层神经网络

加快学习算法的一个办法是随着时间慢慢降低学习率，称为学习率衰减。

如果不进行学习率衰减的话，梯度变化会像蓝线一样，先向全局最小值收敛，然后在其附近波动。

进行学习率衰减则会像绿线一样，刚开始学习率较大，梯度以较快速度下降，后面学习率较小，梯度虽然会在最小值附近波动，但幅度很小。

这个衰减公式将decayrate放在分母，随着迭代数的增加进行衰减。

还有很多学习率下降的方式，比如指数下降、离散下降等。

如果模型的数量少，且训练需要的时间比较长的话，也有人会观察并手动地去调整学习率

一般来说，学习率衰减确实有效，但它不是我们首要去处理的问题。

在机器学习计算指数加权平均的过程中，人们大多不在乎偏差修正，因为大部分人都不关心开始的预测，如果在意早期的估测的话，可以加上偏差修正。

3.1 调试处理

发表于 2023-10-26 分类于 吴恩达深度学习2-改善深层神经网络

在改善神经网络时，你可能需要去调整许多参数

在这些参数中，最重要的是学习率α，其次是β，hidden units和mini-batch size
，第三重要的是layers和learning rate decay，一般不会去调整beta1，beta2和epsilon。

在调整超参数时，传统机器学习的做法是规则取值，而在深度学习中，我们更建议随机取值

在我们不知道哪个参数更重要时，随机取值的效果更好，比如有两个参数α和epsilon，我们现在知道α很重要而epsilon无关紧要(刚开始我们不知道)，如果用左边的训练方式，我们只尝试了5个不同的α值，而如果随机取值，我们会尝试更多α的值，也就更容易取得更好的效果。

调整超参数时也常用从粗略到精细的策略，我们先在整个范围内随机取值，然后看哪个点效果最好(一般这个点附近的值效果也比较好)，然后我们可以以这个点为中心划分更小的区域，在小区域内进行更精细的取值
