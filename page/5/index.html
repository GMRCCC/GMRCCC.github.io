2.4 在不同的划分上进行训练并测试

发表于 2023-10-27 分类于 吴恩达深度学习3-机器学习（ML）策略

深度学习算法对训练数据的胃口很大，这导致很多团队想尽一切方法收集数据，即使数据来自和开发集、测试集不同分布

还是识别猫，假设你有20万张网上下载的高清图片，和1万张模糊一点的图片，识别模糊的图片是你的目标(因为用户上传的图片很可能是模糊的)。

一种训练方法是将这两种图片混合在一起随机划分，保证训练、验证和测试集来自同一分布。但这样做有个坏处，假设验证集的图片有2500张，则其中有近2400张是高清图片，那么验证的效果很可能会偏离我们想达到的目标。

我们可以采用第二种方法，让验证集和测试集全部用模糊的猫图片，保证”瞄准的靶子”是我们想要的。

另一个语音识别的例子，假设你要识别导航的语音，你有50万条来自其他领域的语音数据，和2万条专门针对导航的语音数据，和识别猫类似，验证集和测试集可以全部用专门的语音，训练集则用其他语音和专门语音的混合。

让训练集和开发\测试集不同分布，可以让训练集变得更大，更有利于我们训练模型。

2.6 解决数据不匹配

发表于 2023-10-27 分类于 吴恩达深度学习3-机器学习（ML）策略

数据不匹配的问题没有系统的解决方案，但我们可以做一些尝试。

可以从错误分析中了解训练集和开发测试集的具体差异。

假设你正在开发一个语音激活的后视镜应用，你可能需要听一下验证集的样本来弄清楚开发集和训练集有什么不同。比如你发现相比其他语音后视镜应用语音有很多汽车噪音、经常识别不出街道号码等问题。

知道了问题后，你就可以让训练集和验证集的数据变得更像，比如你可以在训练集的语音中加入模拟汽车噪声的语音，或者有意识地收集人们说数字的语音

这种人工收集的方法不能保证百分百改善数据不匹配的问题，但在很多情况是有用的。

为了让训练集和验证集的数据更相似，我们可以人工合成数据(比如加入模拟的汽车噪音)

当然实际上可能没这么简单，比如还需要加入混响等。但通过这种方法，你可以让训练的数据和验证集的数据更加相似。

这样做有一个问题，假设合成语音中的人声部分时长为1万小时，而汽车噪音部分只有1个小时，那么我们就需要将噪音部分重复1万次和人声部分进行叠加。这在人耳听起来可能没什么问题，但可能会造成噪音部分的过拟合，因为1小时的汽车噪音可能只是所有汽车噪音中的很小一部分

另一个例子是识别汽车，人们有时候会想为什么不直接用计算机合成的图片去训练模型，或者在一个非常逼真的游戏中获得大量的汽车图像

同样的，虽然人眼识别不出来这些汽车，但拟合的图片可能只占所有汽车图片很小一部分，这会导致模型产生过拟合现象。

总之，如果发生数据不匹配问题，我们可以进行错误分析，观察训练集和验证集的数据有哪些不同点，然后看是否能收集更多和验证集相似的数据进行训练。人工合成数据是一个可行的方法，但一定要小心不要过拟合。

2.5 不匹配数据划分的偏差和方差

发表于 2023-10-27 分类于 吴恩达深度学习3-机器学习（ML）策略

当你的训练集和验证\测试集来自不同分布时，分析偏差、方差的方法可能不一样。

如果训练集和验证集来自同一分布，当训练的错误率很小而验证的错误率很大时，我们很清楚是方差出现了问题

当训练集和验证集来自不同分布时，有两件事变了，一是算法只见过训练集数据，没见过验证集数据，二是验证集数据来自不同分布。那么你就很难确认这增加的9%错误率是来自算法没看到开发集中的数据导致，还是因为开发集的数据就是不一样的。

因此我们从训练集中划分出一部分，作为训练-验证集，这部分数据不用来训练，而是用来验证，并且排除了训练集和验证集来自不同分布的问题

我们可以很清楚的看出训练-验证集上的错误率远高于训练集的错误率，因此我们知道问题出在方差上。

第二种情况，训练集和训练-验证集的错误率相差不大，而和验证集的错误率相差较多，我们也知道问题出在数据不匹配上(data mismatch)。

左边很明显发生了偏差问题，右边则是偏差和数据不匹配问题都存在。

有时候，你可能会看到上图右边的情况，训练和训练-开发集上的错误率很高，但验证集\测试集上的错误率反而变低了。当你开发\测试集上的分布比你应用实际处理的数据容易的多时，这种情况是可能出现的。

这时候就需要更加普适的分析

红框划出的是我们经常使用的分析方法，而右上角的两个数据也能让我们洞察出某些特征。比如比较第一行的数据，我们可以知道这个方向要识别的数据比普通的数据要难的多。

2.7 迁移学习

发表于 2023-10-27 分类于 吴恩达深度学习3-机器学习（ML）策略

假设你有一个识别猫的神经网络，现在你想用它来识别X放射图，这种转换就叫做迁移学习

如果你的X放射图数据很小，你可以将识别猫的神经网络拿过来，初始化最后一层的权重，保持其他层的权重不变，对X放射图数据重新训练。如果数据集很大，你也可以对所有层重新训练。

如果你重新训练所有层，那么在猫数据集上训练的过程叫做 预训练(pre-training) ，在X放射图上训练的过程叫做 微调(fine tuning) 。

为什么这样做有用呢？因为在猫数据集上学习的很多低层次特征比如边缘检测、曲线检测、阳性对象检测的能力可能有助于算法在X放射图判断中做的更好。

即识别图像的有些东西是通用的，比如线条、点、曲线的知识，图像的组成和对象的一小部分等等，这些知识可以帮助你的放射图识别网络学习的更快或需要更少的学习数据。

另一个例子是语音识别，假设你有一个听音写字的模型，现在你想把它用来做唤醒词模型。同样的，我们把最后一层去掉，然后加入新的输出节点。有时候我们也可以加入不止一个新节点来训练数据

那么迁移学习有什么意义呢？假设迁移来源问题你有很多数据，但迁移目标问题你没有那么多的数据，那么我们可以借助迁移学习来学习那些共同的特征。

如果反过来，来源问题的数据很少但目标问题的数据很多，那么迁移学习就没有太大意义。

则可以总结出迁移学习有意义的情况为

  1. 两个任务有着同样的输入，比如都是音频，或者都是图像
  2. 当迁移来源的数据远多于迁移目标的数据时，意义更大
  3. 迁移来源任务的低层次特征可以帮助迁移目标任务的学习

2.8 多任务学习

发表于 2023-10-27 分类于 吴恩达深度学习3-机器学习（ML）策略

迁移学习是串行的，神经网络必须先学习任务A，再去学习任务B，而多任务学习则希望能够同时学习多个任务。

假设你有一个无人驾驶任务，你希望汽车能检测行人、汽车、停车标志和信号灯等。

那么你输入图像x，你将会得到一个四维的输出向量y。

如果你有m个样本，那么输出矩阵Y将会是一个(4,m)的矩阵

这就是一个典型的多任务学习神经网络，它不同于学习单个任务的网络(如猫分类器)，输出的向量是多维的，也不同于softmax回归，能同时预测多个标签(softmax虽然也输出多维向量，但它只预测一个标签，即多选一，而多任务学习是多选多)。

多任务学习还有一个特殊点是它能学习只带部分标签的图片，比如一张图片只标注了行人和汽车，而没说明是否有停车标志和信号灯，多任务学习依然能够进行学习。在计算损失函数时，多任务学习模型会跳过未带标签的任务，只计算带标签任务的损失

那么多任务学习什么时候有意义呢？

  1. 训练的一组任务可以共用低层次的特征
  2. 每个任务的数据量很接近(这个条件不绝对)，这样不同任务的学习才能相互促进改变性能
  3. 可以训练一个足够大的网络足以做好所有的任务

多任务学习的代替方法是每个任务都用单个神经网络进行学习，研究发现，多任务学习比单个任务学习会降低性能的唯一情况是你的网络还不够大，如果能够训练一个足够大的神经网络，那么多任务学习的性能往往比一个一个学习的效果要好。

在深度学习界，迁移学习的使用频率比多任务学习高的多，一种可能的原因是你经常要训练小数据量的任务(这意味着需要去借鉴其他数据量大的任务)，但很难找到多个数据量相近的任务。多任务学习最显著的应用是物体检测。

2.9 什么是端到端的深度学习

发表于 2023-10-27 分类于 吴恩达深度学习3-机器学习（ML）策略

端到端的深度学习就是把需要多阶段处理的系统用单个神经网络处理。

以语音识别为例，传统的语音识别需要经过特征提取、音位提取、音位组合成词、词组成文本等过程，而端到端的深度学习只需要一个神经网络就能直接对语音进行识别

这对常年工作在传统方法的某一阶段的科学工作者来说是一件难以接受的事情。

端到端深度学习的一个挑战是必须要喂大量的数据才能有良好的效果，在数据量小的时候，传统方法往往有更好的效果，但数据量一大，端到端深度学习的表现就会超过传统方法了。

当数据量适中时，你也可以采用折中的方法，比如直接用深度学习进行音位的提取，后面的步骤还是遵循传统方法

另一个例子是面部识别，比起端到端的直接识别，业界更常用的方法是分成两步，第一步先在图像中找到人脸，第二步将人脸放大居中进行识别，识别的方法是将人脸与数据库中的人脸进行比较，看是否为同一个人

为什么这里不用端到端的学习而是分成两步呢，有两个原因。一个是要解决的两个问题，每个问题实际上要简单得多，第二个是两个子任务的训练数据都很多(业界领先的公司可能会有上亿张人脸图片)。

相对来说，一步到位的学习就没有那么多数据了(从门禁拍摄的照片对应到具体的人的数据)，所以比起纯粹的端到端，分成两步能够更好的进行学习。

端到端表现比较好的一个例子是机器翻译，传统的机器翻译方法需要经过许多步骤(比如先单个词单个词翻译，再将它们串起来)，但由于我们有大量的数据(从一种语言翻译到另一种语音的数据数不胜数)来进行学习，因此端到端深度学习的效果会比传统方法更好

最后一个例子是通过X射线图去判断一个孩子的年龄(医生们通常用来判断孩子的发育正不正常)，传统做法是先把图片中的一节一节骨骼分割，然后和平均骨长比较长度，这种做法的效果非常好。而端到端的学习，由于数据量的缺乏，目前为止还取不到更好的效果

1.1 计算机视觉

发表于 2023-10-27 分类于 吴恩达深度学习4-卷积神经网络

CV有很多方向的应用，比如图片分类、目标检测、图片风格迁移等

CV的一个问题是输入的数据量可能过于巨大，假设你有一张1000×1000分辨率的图片，那么输入向量就高达3百万，如果第一层有1000个隐藏单元，权重矩阵的参数将高达3亿。这么庞大的参数量意味着我们很难有足够多的数据去防止模型发生过拟合，通常也没有足够大的内存去保存。

此时，我们就需要用到卷积。

1.11 为什么使用卷积？

发表于 2023-10-27 分类于 吴恩达深度学习4-卷积神经网络

和只用全连接层相比，卷积层的两个主要优势在于参数共享和稀疏连接。

如果我们用6个5×5的卷积核来卷积图像，则卷积前后的特征数分别为3072和4704，需要的参数仅为156个，但如果我们想用全连接层来做这个事情，参数个数就变为了3072×4704约14m个，如果图像更大一些，我们可能需要上亿个参数。

参数这么少的原因主要有两个：

  1. 参数共享，图像的不同区域使用同一个卷积核来提取特征，一张图像中的分布往往是相同或者类似的，因此用同一卷积核提取很有效。比如垂直边缘检测器，它即适用于图像的左上角，也适用于图像的右下角和其他任何地方。
  2. 稀疏连接，相比于全连接层输出的特征值用到了输入图像的每个特征值，卷积层输出的特征值只使用了输入图像的一部分特征(和卷积核大小相同)，舍弃了其他不那么相关的特征值。

由于卷积神经网络的参数量更少，所以我们可以用更小的训练集去训练它来减少过拟合。

卷积神经网络善于捕捉平移不变。即使移动几个像素，图像依然具有非常相似的特征，应该属于同样的输出标记。而卷积神经网络使用同样的过滤器在图像上”扫描”来输出像素值，很擅长捕捉偏移的特征

总结来说，卷积神经网络就是利用卷积层、池化层、全连接层、softmax等手段将输入图像抽象为我们想要的输出，在训练过程中通过反向传播去调整各个层的参数以使其有更好的效果

1.2 边缘检测示例

发表于 2023-10-27 分类于 吴恩达深度学习4-卷积神经网络

之前说过，深度神经网络的前几层学习低层次的简单特征，随着层数的深入，学习的东西也会越来越复杂，这些都是通过检测边缘实现的

如果我们要检测一张图片中有什么物体，那么你做的第一件事可能是进行垂直边缘检测，然后进行水平边缘检测

那么如果进行边缘检测？卷积

卷积的过程并不难理解，中间这种矩阵的卷积核叫做垂直边缘检测器。

  • 在不同框架中卷积的表达方式不同，tensorflow中是tf.nn.conv2d，在keras是Conv2D。

为什么垂直边缘检测器长这样呢？

假设我们有一个6×6的图像如左边所示，在用垂直边缘检测器进行卷积后，可以看到在中间有一条很明显的垂直边缘。这条边缘显得比原来的图片粗很多，这是因为原图像的像素太小，如果是1000×1000的图片进行卷积，那么将会很好的得到垂直边缘检测的结果。

从卷积核本身来理解，左侧为1右侧为-1，当图像左右亮度差很多时得到的值会较大，因此能很好地检测垂直边缘。

1.3 更多边缘检测的内容

发表于 2023-10-27 分类于 吴恩达深度学习4-卷积神经网络

垂直边缘检测器可以帮助我们区分边缘是从亮到暗还是从暗到亮

显而易见的，从亮到暗得出的是正值，而从暗到亮得出的是负值。如果你不关心这个，你可以取绝对值来做单纯的边缘检测。

同样的，我们也可以进行水平边缘检测

垂直边缘检测器的数字并不是固定的，历史上曾出现过好几种滤波器，比如Sobel滤波器、Scharr滤波器等，它们会有更好的鲁棒性

当然，你也可以设置9个参数，利用反向传播去计算最好的滤波器形式，使你的图像有更好的边缘检测，不同于单纯的垂直或水平边缘检测，你可以获得任何角度的边缘检测器

通过反馈机制，我们能够让神经网络去学习一些低层次的特征(比如边缘)。但构成这些计算的基础仍然是卷积，通过卷积运算和反向传播我们能调试不同的滤波器，去得到我们想要的一些图片中的特征。
