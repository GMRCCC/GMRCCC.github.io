1.4 满足和优化指标

发表于 2023-10-27 分类于 吴恩达深度学习3-机器学习（ML）策略

有时候把所有因素转化为单个实数指标并不容易，设置满足和优化指标也很有用。

假设我们要识别猫的图片，我们已经有了准确率和运行时间。如果用单一实数指标来评价，则我们可以将这两个指标线性组合，但这样会显得太刻意且实际效果并不好。

事实上，只要运行时间低于100ms，那么用户并不会太在意你实际运行了多少时间，因此我们可以把运行时间当做满足指标，把准确率当做优化指标

另一个例子是关键词触发，比如苹果你希望通过Hi sirri去唤醒助手，那么你需要考虑两个指标，唤醒准确率和误触率，当然也可以综合这两个指标来达到最大的概率值，但更常见的做法是将误触率当做满足指标(比如24小时内只能误触一次)，然后将准确率当做优化指标(越高越好)

1.5 训练_开发_测试集划分

发表于 2023-10-27 分类于 吴恩达深度学习3-机器学习（ML）策略

验证集和测试集最好要来自同一分布，不然你所做的工作可能是无用功。

训练类似于瞄靶，而你所做的训练都是在瞄验证集的’靶心’，如果测试集和验证集是不同分布，相当于测试集的’靶心’和验证集不一样，那么测试结果可能会不尽人意

另一个例子是贷款申请，假设你所训练的模型针对的是中收入人群，忽然有一天领导跟你说要在低收入人群上试试效果，但这两个人群的数据分布很不一样，这可能导致你花几个月训练的模型几乎起不到用处，你必须退回去重新训练

在选择验证集和测试集的数据时，很重要的一点是你未来期望对什么样的数据进行预测，选择类似的数据，而且最好保证验证集和测试集来自同一分布

设立开发集和评估指标，就定义了你要瞄准的目标，而设立训练集则决定了你逼近那个目标有多快。

1.6 开发集和测试集的大小

发表于 2023-10-27 分类于 吴恩达深度学习3-机器学习（ML）策略

传统的训练测试集划分一般不会使测试集的比例低于20%，但随着数据量越来越大，低比例的测试集和验证集也越来越合理

测试集设计的越大，它评估你模型性能的置信度也就越高，如果你的系统不需要那么精确的评估，也就不需要很大的测试集

当你的开发集足够大不会发生过拟合时，没有测试集也不是完全不合理的，当然一般不建议。

1.8 为什么是人的表现

发表于 2023-10-27 分类于 吴恩达深度学习3-机器学习（ML）策略

当你训练一个模型时，当它的效果没有超过人类识别水平时，训练效率会非常高，超过之后，可能效果还能更好但训练效率会急剧下降。最终会收敛于 贝叶斯错误率 ，即从x到y映射的最优函数。

正确率不可能达到100%，比如你识别一段语音，有的语音实在是太过模糊以至于无法识别说的是什么，比如识别猫图片，有些图片分辨率实在是太低以至于无法识别，这就是贝叶斯错误率从宏观意义上的理解。

为什么超过人识别效果后训练效率会变慢，一个原因是人的识别本身就已经接近贝叶斯错误率，超过之后本身就没有太多的改善空间，另一个原因是超过人识别效果后，有些本来好用的工具就会变得没那么好用了。

当算法效果不及人时，人可以继续给数据做标注喂给模型，可以自己判断错误出在哪里，也可以更好的分析偏差和方差；而当算法效果超过人时，这些手段都受到了限制，这导致算法的识别效率下降。

1.7 什么时候该改变开发_测试集的指标

发表于 2023-10-27 分类于 吴恩达深度学习3-机器学习（ML）策略

设置开发集和评估指标相当于设置一个目标靶，训练则是逼近这个靶，但有时候训练到一半我们可能发现目标设置错了，此时要移动你的目标。

假设你要设别猫图片,A分类器的错误率比B分类器的错误率更低，但它可能会设别色情图片为猫图。从评估指标来看，明显A更优秀，但从实际使用情况来说，明显B更合适。此时你的评估指标无法正确衡量两个分类器的优劣，这意味着你需要重新定义评估指标。

关键在于，当发现你对设置的评估指标不满意时，就不要沿用你不满意的错误率指标，而是应该尝试定义一个新的指标，更能够定义出合适的算法。

评估指标本质上可以分成两步，设定靶子(想要达到什么样的效果)和瞄准射击靶子(怎么实现)，这本身就是一个正交化的过程

假设你要设别猫，在训练时你所用的验证集都是从网上下载的高清猫图片，而实际使用时分辨的可能是模糊的猫图片，则设别的效果可能很差。这就是评估指标和实际要达到的目标不匹配，此时你应该去改变你的验证集以匹配你的目的

设置评估指标和开发集可以加快模型的迭代速度，即使你当前无法确定很完美的评估指标，也可以先确定一个能用的指标跑起来，在后续有很好的想法时再进行替换。最好不要在没有评估指标和开发集的数据上训练太长时间，可能会达不到预期的效果。

1.9 可避免误差

发表于 2023-10-27 分类于 吴恩达深度学习3-机器学习（ML）策略

误差一般指的是错误率，而可避免误差就是错误率和人类识别错误率(约等于贝叶斯错误率)的差值

2.1 进行误差分析

发表于 2023-10-27 分类于 吴恩达深度学习3-机器学习（ML）策略

错误分析：如果你希望你的算法能够胜任人类能做的任务，但它现在还未达到人类的表现，人工检查算法犯的错误以知道接下来该做什么的过程叫错误分析。

假设你有一个猫分类器，它的正确率是90%，现在你希望改进这个猫分类器。

你发现在识别错误的图片中有一部分是狗的图片，现在你希望知道是否值得花几个月时间专门对狗图片做算法优化(比如训练更多的狗图片、在算法层面进行改进等等)以减少识别错误率。

错误分析流程：

  1. 从验证集中取约100张识别错误的例子
  2. 数一下其中有多少张是狗的图片

如果其中只有5%是狗的图片，这意味着无论你在识别狗的方向做多少优化，错误率也只能从10%下降到9.5%，这意味着你在这个方向花时间并不划算。

这个优化的上限我们称为”ceiling”，即性能上限。

如果错误的图片中有50%是狗的图片，意味着对狗图片进行优化能让错误率从10%降到5%，那么这个方向就值得一试。

有时候我们可能会排斥人工操作或使用太多人为数值，但如果要搭建应用系统，这个简单的人工错误分析，可能可以节省大量时间来决定哪个方向是最值得优化的。

2.10 是否要使用端到端的深度学习

发表于 2023-10-27 分类于 吴恩达深度学习3-机器学习（ML）策略

端到端学习的好处：

  1. 可以让神经网络自由的学习自己想学的东西，而不是引入人类的成见，一个典型的例子是语音识别，传统的机器学习方法会通过音位去识别，但强迫机器通过音位去识别相当于把人类自己的方法强加给机器，可能无法达到最好的效果。
  2. 所需要的手工组件更少，可以简化工作流程。

端到端学习的缺点：

  1. 需要大量的数据
  2. 排除了可能有用的手工组件，机器获取知识的两个方式是数据和组件，当你的数据足够多时，组件显得不那么重要，但当你的数据不够时，组件也能起到辅助的效果，它可以将人类的许多知识直接灌输到系统里

  • 手工组件是一把双刃剑，当它灌输错误的知识时会起到反效果，但灌输正确的知识可以帮助系统更好的工作。

什么时候使用端到端深度学习？

看你有没有足够的数据完成x到y的映射(学习任务)

就目前来说，端到端的学习前景不如多步学习，一是数据量并未达到预期，二是深度学习并不能学所有的数据类型，只能学有限的几种。

以无人驾驶为例，传统的学习方法是通过传感器的数据识别车和行人，这一步是用深度学习完成的，之后的路径规划和决策控制都不属于深度学习的范围了。而从传感器数据直接到决策控制，还有很长的路要走。

2.2 清除标注错误的数据

发表于 2023-10-27 分类于 吴恩达深度学习3-机器学习（ML）策略

标注的数据一般有输入标签X和输出标签Y，有时候我们会发现标注错误，那是否值得花时间去修正标签？

深度学习算法对随机误差很健壮，所以如果标注错误离随机误差很近(比如标注者不小心标错了)，那么不对错误标签进行改进也不会对训练结果产生太大的影响

深度学习算法对系统性错误的健壮性很差，如果标注者一直把白色的狗当做猫，那么训练出来的分类器也会把白色的狗当做猫，但随机或近随机错误不会有太大影响。

上述讲的是在训练集中标注错误的情况，如果在开发/测试集中标注错误该怎么办？

我们可以在错误分析中额外加上一列来统计标签Y错误的例子数

如果标签标注错误没有严重影响到用开发集评估成本偏差的能力，那么可能不应该花费宝贵的时间去处理。

也可以看三个指标，整体的开发集错误率、错误标记引起的错误数量或百分比和其他原因造成错误的数量或百分比

可以看到由其他原因造成的错误率为9.4%，此时应该集中精力处理9.4%的错误，把标注错误先放一放。

当你把整体错误率优化到2%时，标注错误引起的错误率0.6%就显得很重要，需要去优化了

验证集的一个重要作用是评估两个分类器的优劣，如果A分类器的错误率是2.1%，B分类器的错误率是1.9%，它们的标注错误率为0.6%，此时就很难评估两个分类器的好坏，我们需要去修正标注错误来重新进行评估。

在修正标注时，我们最好将验证集和测试集一起修正，以保证它们还是同一分布。

第二点我们也需要考虑判断正确例子中有无标注错误，由于判断错误的例子占比较小所以修正判断错误例子中的错误标注较为简单，但就算在判断正确的例子中寻找标注错误较难也是要考虑到的。

最后，当我们修正开发/测试集的标注时，我们可以不那么去考虑训练集中的标注错误，一个是随机错误并不那么影响算法的健壮性，另一个是虽然开发/测试集需要是同一分布，但训练集可以有稍微不同的分布。

2.3 快速搭建你的第一个系统，并进行迭代

发表于 2023-10-27 分类于 吴恩达深度学习3-机器学习（ML）策略

如果你在开发全新的机器学习应用，你应该尽快建立你的第一个系统原型，然后快速迭代。

如你要做语音识别，你有很多方向可以选择

一般来说，对所有的机器学习程序，大概有50个方向可供选择，而且所有方向都是合理的能够改善你的系统，难点在于，如何在刚开始就选择去专精哪一个方向。

所以如果想搭建全新的机器学习程序，建议是快速搭好第一个系统，然后开始迭代，即：

  1. 快速设立开发集、测试集和指标
  2. 快速搭好一个机器学习系统原型，开始训练
  3. 使用偏差/方差分析和错误分析来决定下一步优先做什么

这样做的好处是能够让你快速确定偏差/方差的范围，知道下一步该做什么；能够进行错误分析，知道下一步该往哪个方向走。

如果你做的方向是一个成熟的领域(如人脸识别)，你也可以在大量文献的基础上一开始就搭建复杂的系统。
